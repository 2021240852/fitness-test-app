# 体质健康测试 - 安卓APK构建说明

## 方法一：使用 Android Studio（推荐）

### 准备工作

1. **下载并安装 Android Studio**
   - 访问 https://developer.android.com/studio
   - 下载并安装最新版 Android Studio

2. **安装必要组件**
   - 打开 Android Studio
   - 进入 SDK Manager (Tools > SDK Manager)
   - 安装以下组件：
     - Android SDK Platform 33（或更高版本）
     - Android SDK Build-Tools
     - Android Emulator（可选，用于测试）

### 构建步骤

1. **解压项目文件**
   ```
   解压 android-project.tar.gz 到任意文件夹
   ```

2. **用 Android Studio 打开项目**
   - 打开 Android Studio
   - 选择 "Open an existing project"
   - 选择解压后的 `android` 文件夹

3. **等待 Gradle 同步**
   - Android Studio 会自动下载依赖
   - 等待同步完成（可能需要几分钟）

4. **构建 APK**
   - 点击菜单栏 `Build` > `Build Bundle(s) / APK(s)` > `Build APK(s)`
   - 或者使用快捷键：Ctrl+F9 (Windows/Linux) 或 Cmd+F9 (Mac)

5. **获取 APK 文件**
   - 构建完成后，点击右下角的提示 "Show in Explorer"
   - APK 文件位置：`android/app/build/outputs/apk/debug/app-debug.apk`

### 安装到手机

1. **启用开发者模式**
   - 手机设置 > 关于手机 > 连续点击"版本号"7次
   - 返回设置 > 开发者选项 > 开启"USB调试"

2. **连接手机并安装**
   - 用 USB 线连接手机和电脑
   - 在 Android Studio 中点击 `Run` > `Run 'app'`
   - 或直接将 APK 文件复制到手机安装

## 方法二：使用命令行（适合高级用户）

### 准备工作

1. **安装 Java JDK 17**
   ```bash
   # 检查是否已安装
   java -version
   ```

2. **安装 Android SDK**
   - 下载 Android Command Line Tools
   - 设置环境变量：
     ```bash
     export ANDROID_SDK_ROOT=/path/to/android-sdk
     export PATH=$PATH:$ANDROID_SDK_ROOT/cmdline-tools/latest/bin
     ```

### 构建步骤

1. **进入安卓项目目录**
   ```bash
   cd android
   ```

2. **赋予 Gradle 执行权限**
   ```bash
   chmod +x gradlew
   ```

3. **构建 Debug APK**
   ```bash
   ./gradlew assembleDebug
   ```

4. **构建 Release APK**
   ```bash
   ./gradlew assembleRelease
   ```

5. **获取 APK 文件**
   - Debug 版本：`app/build/outputs/apk/debug/app-debug.apk`
   - Release 版本：`app/build/outputs/apk/release/app-release-unsigned.apk`

## 常见问题

### Q: 构建时出现 "SDK not found" 错误？
A: 确保已安装 Android SDK 并正确设置 ANDROID_SDK_ROOT 环境变量

### Q: Gradle 同步失败？
A: 检查网络连接，或尝试更换 Gradle 镜像源

### Q: 安装时提示 "应用未安装"？
A: 可能是签名问题，尝试构建 debug 版本或创建签名密钥
